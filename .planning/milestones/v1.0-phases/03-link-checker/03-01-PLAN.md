    RED phase -- write these tests first (all must fail initially):

    checkUrl status resolution:
    - checkUrl('https://ok.com') with mocked 200 response returns { status: 'ok', finalUrl: 'https://ok.com' }
    - checkUrl('https://dead.com') with mocked 404 response returns { status: 'dead' }
    - checkUrl('https://timeout.com') with mocked AbortError returns { status: 'dead' }
    - checkUrl('https://dns.com') with mocked ENOTFOUND error returns { status: 'dead' }
    - checkUrl('https://ratelimit.com') with mocked 429 response returns { status: 'uncertain' }
    - checkUrl('https://auth.com') with mocked 401 response returns { status: 'ok' }
    - checkUrl('https://forbidden.com') with mocked 403 response returns { status: 'ok' }
    - checkUrl('https://nohead.com') with HEAD returning 405, GET returning 200 returns { status: 'ok' } (per D-09)
    - checkUrl('https://nohead.com') with HEAD returning 405, GET returning 404 returns { status: 'dead' }
    - checkUrl('not a valid url') returns { status: 'dead' } (malformed URL, per Pitfall 6)
    - checkUrl('https://dead-head.com') with HEAD returning 404 does NOT attempt GET (non-2xx HEAD is definitive)

    HEAD-first with GET fallback:
    - HEAD is attempted first with AbortSignal.timeout(5000) (per D-13)
    - GET fallback uses AbortSignal.timeout(8000) (per D-14)
    - GET fallback triggered ONLY on 405 or thrown error, NOT on non-2xx status codes

    OG metadata extraction (per D-16, D-17, D-18, D-19):
    - When GET returns 2xx with Content-Type text/html, metadata object contains og:title, og:description, og:image, meta description
    - When HEAD returns 2xx (no body), metadata is null/undefined
    - When GET returns 2xx with Content-Type application/json, metadata is null/undefined (not HTML)

    buildCheckedTree (per D-03, LINK-02):
    - Removes nodes where linkStatus === 'dead'
    - Keeps nodes where linkStatus === 'ok'
    - Keeps nodes where linkStatus === 'uncertain'
    - Preserves folder structure (empty folders kept -- CLASS-03 is Phase 5)
    - Does NOT mutate the input tree (immutable-stage pattern)

    checkAll with onProgress:
    - Calls onProgress for each completed URL with { checked, total, currentUrl, eta }
    - Returns { checkedTree, deadCount, uncertainCount }
    - Uses two-level concurrency: global limit 20, per-domain limit 2 (per D-11)

    Redirect capture:
    - When response.url differs from input URL, result includes finalUrl field
  </behavior>

    - `checkAll(tree, onProgress)` -- walk tree, collect link nodes, check all with two-level p-limit concurrency, build checkedTree

    Internal functions:
    - `getDomainLimiter(hostname)` -- lazy Map<string, pLimit(2)> per D-12
    - `_fetchUrl(url)` -- HEAD attempt (5s timeout), GET fallback on 405/error (8s timeout)
    - `resolveStatus(response, originalUrl)` -- 429 -> 'uncertain', 401/403 -> 'ok', 2xx -> 'ok', else -> 'dead' (per D-05 through D-08)
    - `extractMetadata(response)` -- check Content-Type includes 'text/html', parse body with cheerio, extract og:title, og:description, og:image, meta[name=description] (per D-16, D-17, D-19)
    - `buildCheckedTree(tree, linkStatuses)` -- deep clone tree, remove dead nodes, annotate surviving nodes with linkStatus and metadata (per D-03)
    - `collectLinks(tree)` -- recursive walk collecting { url, nodeId } for type === 'link' nodes
    - `calcEta(checked, total, elapsedMs)` -- linear extrapolation: remaining = (total - checked), rate = checked/elapsedMs, eta = remaining/rate/1000 seconds

    Two-level concurrency setup (per D-10, D-11, D-12):
    ```javascript
    const globalLimit = pLimit(20);
    const domainLimiters = new Map();
    function getDomainLimiter(hostname) {
      if (!domainLimiters.has(hostname)) domainLimiters.set(hostname, pLimit(2));
      return domainLimiters.get(hostname);
    }
    ```

    Each URL wrapped: `globalLimit(() => domainLimit(() => _fetchUrl(url)))`

    Malformed URL handling (Pitfall 6): wrap `new URL(url)` in try/catch, return { status: 'dead' } immediately if it throws.

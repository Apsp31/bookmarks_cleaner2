## Interaction Contract

### "Run Cleanup" Flow

1. Button appears in `.actions` bar when `status === 'loaded'`. Label: **"Run Cleanup"**.
2. On click: status transitions to `'cleaning'`. Spinner shown. Button disabled.
3. POST `/api/cleanup` called. Response includes `cleanTree`, `stats`, `mergeCandidates[]`.
4. On success: status transitions to `'cleaned'`.
   - `.cleanup-banner` shown: "Cleanup complete — X duplicate bookmarks removed."
   - If `mergeCandidates.length > 0`: tree re-renders in merge-review mode (⚠️ badges inline).
   - If `mergeCandidates.length === 0`: tree renders clean with no badges; bulk approve button absent.
5. On error: status transitions to `'error'`. `.error-msg` shown.

### Merge Review Mode

- `renderTree()` called with a `reviewMode: true` flag (second argument or options object).
- For each folder node whose ID appears in `mergeCandidates[]`:
  - Append `.merge-badge` (⚠️) after the folder label `<span>`.
  - Append `.btn-merge` labeled `→ Merge into "[target name]"`.
  - Append `.btn-keep` labeled `Keep separate`.
- Bulk approve bar: rendered above the tree panel when `mergeCandidates.length > 0`. Single `.btn-approve-all` button. On click: calls POST `/api/merge` with `{ approveAll: true }`.
- Per-row merge: `.btn-merge` click calls POST `/api/merge` with `{ pairs: [{ a: id, b: id }] }`. Tree re-renders. Badge and buttons removed from resolved row.
- Per-row keep: `.btn-keep` click removes badge and buttons from that row only (client-side). No API call required.

### Duplicate Subtree Confirmation

- Identical inline pattern to folder merge — uses `.merge-badge`, `.btn-merge` ("Remove duplicate"), `.btn-keep` ("Keep both") buttons inline per flagged folder pair.
- Bulk approve includes subtree removals in same API call.

### Export after Cleanup

- "Export Clean File" button always present when `status === 'loaded' || status === 'cleaned'`.
- When `session.cleanTree` exists (post-cleanup), export route serves `cleanTree`.
- No change to button label or placement.

## States and Transitions

| State | Status value | What is shown |
|-------|-------------|---------------|
| No file | `idle` | Drop zone + file picker |
| File invalid | `error` | Drop zone + `.error-msg` |
| Uploading/parsing | `loading` | Spinner + "Processing your bookmarks…" |
| File loaded, cleanup not run | `loaded` | Stats line + actions bar (Export + Run Cleanup + Load another file) + tree panel |
| Cleanup running | `cleaning` | Actions bar disabled, spinner + "Running cleanup…" |
| Cleanup done, merges pending | `cleaned` | Cleanup banner + bulk approve + tree in review mode + Export button |
| Cleanup done, no merges | `cleaned` | Cleanup banner (clean copy) + tree (no badges) + Export button |
| Merge/keep resolving | `cleaned` (sub-state) | Inline per-row badge/buttons update without full re-render |

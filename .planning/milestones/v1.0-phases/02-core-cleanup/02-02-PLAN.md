**2. Create `src/routes/cleanup.js`:**

Per D-08, POST /api/cleanup returns `{ cleanTree, stats, mergeCandidates, duplicateSubtrees }`.

```javascript
import { Router } from 'express';
import { dedupTree, countLinks } from '../dedup.js';
import { findMergeCandidates, findDuplicateSubtrees } from '../fuzzy.js';
import { session } from '../session.js';

const router = Router();

router.post('/cleanup', (req, res) => {
  if (!session.tree) {
    return res.status(400).json({ error: 'No bookmark file loaded.' });
  }

  // 1. Dedup — returns new tree, never mutates session.tree (per D-07)
  const originalLinkCount = countLinks(session.tree);
  const cleanTree = dedupTree(session.tree);
  const cleanLinkCount = countLinks(cleanTree);
  const dupesRemoved = originalLinkCount - cleanLinkCount;

  // 2. Fuzzy folder matches (per D-06, threshold 0.25)
  const mergeCandidates = findMergeCandidates(cleanTree);

  // 3. Duplicate subtrees
  const duplicateSubtrees = findDuplicateSubtrees(cleanTree);

  // 4. Persist to session
  session.cleanTree = cleanTree;
  session.mergeCandidates = mergeCandidates;
  session.duplicateSubtrees = duplicateSubtrees;

  res.json({
    cleanTree,
    stats: { dupesRemoved, duplicateSubtreesFound: duplicateSubtrees.length },
    mergeCandidates,
    duplicateSubtrees,
  });
});

export default router;
```

Follow the Router-per-file pattern established in Phase 1 (D-08 from 01-CONTEXT.md).

**1. Create `src/routes/merge.js`:**

POST /api/merge accepts either `{ approveAll: true }` or `{ pairs: [{ aId, bId }, ...] }`.

The route must:
- Check `session.cleanTree` exists, return 400 if not
- Determine merge list from `approveAll` (use all `session.mergeCandidates` + `session.duplicateSubtrees`) or from `pairs` array
- For each merge pair `{ aId, bId }`: recursively walk cleanTree to find folder with id `bId`, extract its children, find folder with id `aId`, append children, remove folder `bId` from its parent — all producing a NEW tree (never mutate in place per D-07/D-11)
- Update `session.cleanTree` with the result
- If `approveAll`: clear both `session.mergeCandidates` and `session.duplicateSubtrees`
- If partial: remove resolved pairs from `session.mergeCandidates` where aId or bId matches
- Return `{ cleanTree: updatedTree, mergeCandidates: session.mergeCandidates, duplicateSubtrees: session.duplicateSubtrees }`
- IMPORTANT per Pitfall 3 from RESEARCH.md: response MUST include updated mergeCandidates so frontend stays in sync

Implement `applyMerge(tree, keepId, removeId)` as a helper:
- Recursive function that walks the tree
- When it finds the folder with `removeId`, extract its children
- When it finds the folder with `keepId`, append the extracted children
- Remove the `removeId` folder from its parent's children array
- Return a new tree (shallow copy at each level)

Use `express.json()` middleware inside the router file via `router.use(express.json())` or ensure `server.js` has `app.use(express.json())` before the route mounts. Express 5 does NOT auto-parse JSON — add `app.use(express.json())` to `server.js` after the static middleware line and before the route mounts.

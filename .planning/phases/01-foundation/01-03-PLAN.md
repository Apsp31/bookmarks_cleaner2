POST /api/upload
  Request: multipart/form-data with field name "bookmarks" (HTML file)
  Response 200: { tree: BookmarkNode, stats: { bookmarkCount: number, folderCount: number } }

  Response 200: Netscape HTML file download (Content-Disposition: attachment)
  Response 404: { error: "No bookmark file loaded. Upload a file first." }

BookmarkNode shape:
  { id: string, type: 'folder'|'link', title: string, url?: string, addDate?: number, children?: BookmarkNode[] }

2. Create `public/app.js`:
   - `document.addEventListener('alpine:init', () => { Alpine.data('bookmarkApp', () => ({ ... })); });`
   - State properties:
     - `status: 'idle'` — 'idle' | 'loading' | 'loaded' | 'error'
     - `stats: null` — { bookmarkCount, folderCount }
     - `tree: null` — BookmarkNode root
     - `backupName: ''` — filename of backup download
     - `errorMsg: ''` — error message to display
   - Methods:
     - `handleFile(file)`:
       - Validate: if `!file` or not `.html` extension, set errorMsg and return
       - Set `status = 'loading'`
       - **Trigger backup download FIRST, synchronously, before any await** (per D-05, Pitfall 5 from research — must happen in user gesture context):
         - `this.backupName = 'bookmarks-backup-' + new Date().toISOString().slice(0, 10) + '.html'`
         - Create object URL from file, create `<a>` element, set `href` and `download` attributes, click it, revoke URL
       - Upload to server: `const formData = new FormData(); formData.append('bookmarks', file);`
       - `const res = await fetch('/api/upload', { method: 'POST', body: formData });`
       - On error: set `status = 'error'`, `errorMsg` from response
       - On success: set `tree = data.tree`, `stats = data.stats`, `status = 'loaded'`
     - `onDrop(e)`: call `handleFile(e.dataTransfer.files[0])`
     - `handleFileInput(e)`: call `handleFile(e.target.files[0])`
     - `exportBookmarks()`: `window.location.href = '/api/export';`
     - `resetApp()`: reset all state to defaults
     - `init()`: on component init, call `renderTree` setup (if needed)
   - Tree rendering:
     - Since Alpine.js does not support recursive templates natively (per research), use a vanilla JS function `renderTree(node, container, depth)` that:
       - Creates a `<div>` for each node with left padding based on depth
       - For folders: creates a clickable toggle (arrow icon or +/-) and folder name. On click, toggles visibility of children container. Children start collapsed.
       - For links: shows title as text and URL in smaller/muted text
       - Called from Alpine's `$nextTick` or `x-effect` after tree data loads, targeting a `<div x-ref="treeContainer">`

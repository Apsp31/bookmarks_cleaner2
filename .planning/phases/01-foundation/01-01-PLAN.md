5. Create `src/parser.js`:
   - `import * as cheerio from 'cheerio';`
   - `import { randomUUID } from 'crypto';`
   - Export function `parseBookmarkHtml(html)` that:
     - Loads HTML with `cheerio.load(html, { xmlMode: false })`
     - Finds root `<dl>` with `$('dl').first()`
     - Recursively walks `<dt>` children of each `<dl>`
     - For each `<dt>`: check first child — if `<h3>` it's a folder, if `<a>` it's a link
     - For folders: use `$dt.nextAll('dl').first()` to find child list (NOT `.children('dl')` — Pitfall 2 from research)
     - Parse `add_date` attribute: `parseInt($el.attr('add_date'), 10)` only if attribute exists, else `undefined`
     - Assign `crypto.randomUUID()` as `id` to every node
     - Returns root BookmarkNode with `type: 'folder'`, `title: 'root'`

6. Create `src/exporter.js`:
   - Export function `exportToNetscape(rootNode)` that:
     - Produces exact Netscape format with DOCTYPE, META, TITLE, H1
     - `escapeHtml(str)` helper that replaces `&` -> `&amp;`, `"` -> `&quot;`, `<` -> `&lt;`, `>` -> `&gt;` (in that order — `&` must be first)
     - Recursively serializes nodes with proper `<DT><A HREF="...">` for links and `<DT><H3>` + `<DL><p>` for folders
     - Only includes `ADD_DATE="..."` attribute when `node.addDate` is defined and is a finite number
     - All titles AND URL attribute values go through `escapeHtml()` (Pitfall 1 from research — unescaped `&` breaks Chrome import)

7. Run tests — they MUST pass (GREEN phase).
